Air Pollution Monitoring System

PythonRaspberry PiGas Sensors GSM ModuleBlynk AppIoT

Overview

A real-time IoT-based air quality monitoring system that continuously reads atmospheric gas levels using dedicated sensors. When a dangerous gas concentration reaches or exceeds the safe threshold, the system immediately triggers an alert — sending SMS notifications directly to government officials and safety authorities via a GSM module, and simultaneously pushing live readings to the Blynk mobile app for remote monitoring.

Hardware Used

  • MQ-Series Gas Sensors: Detect harmful gases like CO, CO₂, LPG, Smoke, and NH₃ in the atmosphere in real time.
  • GSM Module (SIM800L): Sends automated SMS alerts to pre-configured emergency contacts when thresholds are breached.
  • Raspberry Pi: Central processing unit that runs the Python monitoring script, reads sensor data, and triggers alerts.
  • Blynk App: Provides a real-time mobile dashboard to visualize gas levels, historical graphs, and receive push notifications.

System Flow

1. Gas Detection

MQ sensors continuously sample atmospheric gas concentrations and output analog voltage values.

2. Data Processing

Raspberry Pi reads sensor values, converts them to PPM units, and compares against safety thresholds via Python.

3. Threshold Check

If gas level exceeds the safe limit, the alert pipeline is triggered immediately.

4. SMS + App Alert

GSM module sends SMS to authorities. Blynk app pushes a real-time notification with gas type and PPM level.

Driver Drowsiness Detection

PythonOpenCVDlibComputer Vision

Overview

A computer vision-based safety system that continuously monitors a driver's face in real-time. By tracking facial landmarks and calculating the Eye Aspect Ratio (EAR), the system accurately detects signs of microsleep or severe fatigue, instantly triggering an alarm to wake the driver and prevent potential accidents.

Devices & Hardware

  • HD Camera: Captures clear real-time video feed of the driver's face.
  • Processing Node: A local processing unit (Laptop/Raspberry Pi) to run the ML inference without latency.
  • Audio Alarm System: Triggers immediate high-frequency alerts.

System Flow Diagram

1. Video Capture

Initialize webcam feed and capture continuous frames.

2. Face Detection

Isolate the driver's face using HOG detectors.

3. Landmark Extraction

Map specific facial points to locate the exact contours of both eyes.

4. EAR Calculation

Compute the Eye Aspect Ratio. If EAR falls below the threshold for consecutive frames, trigger the alarm.
